9Terms & Next StepsTwo-column: terms left, next steps right. Bottom CTA with email.

Hero Section

Every proposal uses hero-canvas.js — a reusable animated constellation/network graphic. The hero must be min-height: 100vh and no other section should be visible on initial load.

Rules

✗No personal names — use "Core Digital Expansion" only
✗No scroll reveal animations — all content visible immediately
✗No pricing in the hero — lead with mission/value
✗No fixed max-width on sections — use padding only
✓Contact email: balaji@cd-expansion.com
✓Dark cards use solid #2a2a2a backgrounds, not transparent rgba
✓Pricing blocks use colored left borders: Blue (one-time), Mint (recurring), Cyan (performance)
✓Always QA with Playwright before pushing
✓Research uses official/primary sources only — no Statista, no aggregators

Lessons Learned

1Don't use position: relative on all children when parent has absolute-positioned decorative elements
2Never use fixed max-width on sections — causes right-side clipping with 260px sidebar
3Dark section cards must use solid backgrounds (#2a2a2a), not rgba(255,255,255,0.06)
4Sidebar active state text: #ffffff with font-weight: 600
5Simple Icons CDN doesn't have all brands — fall back to balajibhat/tech-logos
6Figma community SVG exports include wordmark + border — use icons/ subfolder for icon-only
