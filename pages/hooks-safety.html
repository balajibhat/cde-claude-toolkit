Hooks & Safety

Security hooks protect your machine from accidental damage. Here's what they do and why they exist.

Origin story: These hooks exist because an AI command bricked a Windows installation in March 2026. System registry commands ran unsupervised and corrupted the OS. These hooks make sure that never happens again.

Hook 1: Bash Command Guard

Runs before every terminal command Claude tries to execute. Five layers of protection:

LayerActionExamples
Windows System ToolsDENYreg, dism, sfc, bcdedit, diskpart
Destructive File OpsDENY / ASKrm -rf /, format, mass deletion
Credential ExfiltrationDENY / ASKCurl/wget with credential paths, piping secrets
Remote Code ExecutionDENYInvoke-WebRequest | iex, eval of remote scripts
Git Destructive OpsASKgit push --force, git reset --hard

Hook 2: File Edit Guard

Runs before Claude edits or writes any file. Blocks writes to sensitive locations:

Protected PathActionWhy
credentials/BLOCKAPI keys and tokens must never be modified by AI
.env filesBLOCKEnvironment variables often contain secrets
.claude/hooks/BLOCKSelf-protection — hooks can't disable themselves
settings.jsonBLOCKPrevents permission escalation
Windows system pathsBLOCKNo OS-level file modifications
.claude/ directoryASKPrompts for confirmation before modifying Claude config

What This Means for You

You can safely
Let Claude run commands, edit code, create files in your working directories. Hooks only block dangerous patterns.
If you get blocked
The hook will explain why. If it's a false positive, do the operation manually. Don't try to bypass hooks.
Never disable hooks. If a hook blocks something you think is safe, do it manually or ask Balaji. The hooks are there because something went wrong before.

Setting Up Hooks on Your Machine

Hooks are JavaScript files that live in ~/.claude/hooks/ and are configured in ~/.claude/settings.json. We've included an installer script to set them up automatically.

Option A: Run the Installer (Recommended)

From the cde-claude-toolkit directory you cloned:

# Windows (Command Prompt or PowerShell) starter\hooks\install-hooks.bat # Mac / Linux / Git Bash bash starter/hooks/install-hooks.sh

The installer copies the hook scripts and creates or guides you through updating settings.json.

Option B: Manual Setup

If you prefer to do it by hand:

  1. Create the hooks directory
    # Windows mkdir %USERPROFILE%\.claude\hooks # Mac / Linux mkdir -p ~/.claude/hooks
  2. Copy the hook files
    # Windows copy cde-claude-toolkit\starter\hooks\check-system-commands.js %USERPROFILE%\.claude\hooks\ copy cde-claude-toolkit\starter\hooks\check-file-edits.js %USERPROFILE%\.claude\hooks\ # Mac / Linux cp cde-claude-toolkit/starter/hooks/check-system-commands.js ~/.claude/hooks/ cp cde-claude-toolkit/starter/hooks/check-file-edits.js ~/.claude/hooks/
  3. Add hooks to settings.json
    Open ~/.claude/settings.json (create it if it doesn't exist) and add the "hooks" block from starter/hooks/settings-hooks.json. The key section:
    { "hooks": { "PreToolUse": [ { "matcher": "Bash", "hooks": [{ "type": "command", "command": "node \"$HOME/.claude/hooks/check-system-commands.js\"", "timeout": 5 }] }, { "matcher": "Edit|Write", "hooks": [{ "type": "command", "command": "node \"$HOME/.claude/hooks/check-file-edits.js\"", "timeout": 5 }] } ] } }

Verify It Works

Open Claude Code (VS Code, Desktop App, or CLI) and ask Claude to run any bash command. You should see "Checking for dangerous commands..." flash in the status bar. That means the hook is active.

Works everywhere: Hooks run the same way whether you use VS Code, the Desktop App, or the CLI. They're configured at the user level in ~/.claude/, not per-app.
Requires Node.js. The hook scripts are JavaScript files executed by node. If you don't have Node.js installed, get it from nodejs.org (LTS version).
