Shopping Feed Management Protocol

How CDE builds and maintains Google Merchant Center supplemental feeds. The most powerful lever for Shopping and PMax campaign performance.

Core Principle

A supplemental feed is how CDE controls what Google sees about a client's products. The primary feed (Shopify sync) handles basics — our supplemental feed adds the strategic layer: custom labels, corrected categories, missing attributes.

ID Format Reality

Shopify feeds into GMC using two ID formats for the same variant:

Shopify-prefix: shopify_US_{product_id}_{variant_id} Numeric-only: {variant_id}
Critical: The primary feed may use BOTH formats. Before deduplicating, verify match counts. If matched_count > max(type_a_count, type_b_count), both formats are in play. Removing one drops match rate.

Feed QA Protocol

Before Uploading

  1. Never edit CSVs in Excel — Excel silently corrupts long numeric IDs into scientific notation (42030246297687 → 4.20302E+13). Use VS Code, Notepad++, or Python scripts.
  2. Validate ID integrity — Spot-check that IDs are full-length numeric strings, not scientific notation.
  3. Check column headers — GMC only recognizes specific attribute names. Common mistakes: product_name (should be title), spaces instead of underscores.
  4. Check for "Offer does not exist" errors — These mean supplemental feed has IDs not in the primary feed. Usually discontinued products or ID format mismatches.

After Uploading

  1. Check "Matched products" count — Should be close to your row count. Large gaps = ID mismatches.
  2. Check "Attribute names" — Should say "All recognized." Unrecognized attributes are wasted columns.
  3. Download the error report — Parse the CSV to identify specific problem products.
  4. Cross-reference against primary feed — Are failing products actually in the primary feed?

Custom Label Strategy

Custom labels bridge the feed and campaign structure. Design them BEFORE building campaigns.

SlotRecommended UseWhy
custom_label_0Reserved / Promo flagsClient may use this for sale exclusions — check first
custom_label_1Priority tier (hero / core / long_tail)Bid segmentation by revenue importance
custom_label_2Product categoryCategory-level bid control
custom_label_3Collection / product lineCollection-level segmentation
custom_label_4SeasonalitySeasonal bid adjustments

Priority Tier Classification

HeroTop revenue-driving collections (2-3 lines = 60%+ of revenue)
CoreSolid performers, consistent demand
Long tailLow volume, niche, accessories, gift cards

Feed → Campaign Connection

Every custom label should map to a campaign decision. If it doesn't drive a bid or targeting change, it's wasting a slot.

custom_label_1 = hero → Higher bids, dedicated campaign custom_label_1 = core → Standard bids, grouped campaign custom_label_1 = long_tail → Lower bids or excluded from Shopping custom_label_2 = shorts → Seasonal bid adjustments (summer up) custom_label_2 = outerwear → Inverse seasonal adjustments custom_label_4 = summer_peak → Increase bids May-Aug custom_label_4 = winter_peak → Increase bids Nov-Feb custom_label_4 = evergreen → Stable year-round

Monthly Maintenance

  1. Export fresh Shopify data — new products, discontinued, price changes
  2. Run the supplemental feed script against the new export
  3. Diff against previous month — add new, remove discontinued, update categories
  4. Upload and validate — check match rates, download error report
