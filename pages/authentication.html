Authentication

How to set up API keys and OAuth tokens for the MCP servers that need them.

Security first. Never commit API keys, tokens, or credentials to any repository. Never paste them into a Claude conversation. Store them in your local credentials directory and reference them via environment variables.

Credentials Directory

Create a local credentials directory that is never committed to git:

# Windows mkdir "%USERPROFILE%\Documents\Claude Code\credentials" # Mac mkdir -p ~/Documents/Claude\ Code/credentials
Add this directory to your global .gitignore as an extra safety net. Claude Code's security hooks will also block any attempt to read or commit files from this directory.

Server-Specific Auth

Nano Banana (Gemini API Key)

  1. Go to aistudio.google.com/apikey and create a Gemini API key
  2. Save it to your credentials directory as gemini-api-key.txt
  3. In Claude Code, use the configure_gemini_token tool to set it up, or set the environment variable GEMINI_API_KEY

Google Drive (OAuth)

Google Drive uses OAuth 2.0 — you'll need to authenticate through a browser flow. Ask Balaji for the OAuth client credentials file, then:

  1. Place the gcp-oauth.keys.json file in ~/.claude/
  2. Run the gdrive MCP server — it will open a browser window for Google sign-in
  3. Authorize the app. Tokens are saved locally at ~/.claude/.gdrive-server-credentials.json

CDE Imagen (Gemini API Key)

Uses the same Gemini API key as Nano Banana. Set the GEMINI_API_KEY environment variable or configure it in the server's env settings.

Google Ads (Complex)

Requires OAuth credentials, a developer token, and MCC access. This is admin-level setup — talk to Balaji if you need Google Ads access.

Environment Variables

Some servers read API keys from environment variables. You can set these in your shell profile or in Claude Code's MCP server config:

# In settings.json, you can pass env vars to MCP servers: { "mcpServers": { "cde-imagen": { "command": "node", "args": ["path/to/server.js"], "env": { "GEMINI_API_KEY": "your-key-here" } } } }
If you put keys in settings.json, make sure that file is not synced or committed anywhere. It lives at ~/.claude/settings.json and should stay local.
