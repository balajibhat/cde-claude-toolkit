Day 1 Workflows

Real prompts for real work. Copy these into Claude Code and start getting things done.

How to use this page: Each workflow below is a prompt you can paste directly into Claude Code (VS Code, Desktop App, or CLI). Swap the bracketed placeholders with your actual details.

Research

Competitor Analysis

Uses Playwright to visit competitor sites and Supadata for any YouTube content.

Research [competitor name]'s website at [URL]. I need: - What services/products they offer - How they position themselves (tagline, hero messaging) - Pricing if visible - Any case studies or social proof Use only what's on their actual website. No guessing.

Industry Research

Deep research using primary sources only (per CDE rules).

Research the [industry/topic] market for a client pitch. I need: - Market size (from official industry reports, not aggregators) - Key trends (cite the source for each) - Top 3-5 players and what they do differently Only use official sources — no Statista, no SEO blogs. Cite everything.

YouTube Video Summary

Pulls transcript via Supadata and summarizes it.

Get the transcript from this video: [YouTube URL] Summarize the key points in bullet format. Flag any stats or claims that would be useful for our [client/project] work.

QA & Testing

Full Page QA

Uses Playwright to inspect a live page. Essential before sharing any deliverable with a client.

QA this page: [URL] Check: - All navigation links work (no 404s) - No console errors - Mobile responsive (check at 375px width) - All images load - Text is readable, no overflow or clipping - Forms work if any exist Take screenshots at desktop and mobile widths.

Quick Link Check

Navigate to [URL] and check every link on the page. Report any that are broken or lead to 404s.

Cross-Browser Spot Check

Navigate to [URL] and take a screenshot. Then check: - Does the layout match the design intent? - Any text overflow or alignment issues? - Do all interactive elements (buttons, forms, dropdowns) look correct?

Content Creation

Blog Post from Topic

Write a blog post for CDE about [topic]. Target audience: [who reads this] Tone: professional but conversational, no fluff Length: ~800-1200 words Include: specific examples, data points with citations Remember: no AI filler words (check the Writing Standards protocol).

Social Media Copy

Write 3 LinkedIn post options for [topic/announcement]. Each post should: - Have a strong hook in the first line - Be under 200 words - End with a clear CTA - Sound like a real person, not a corporate account - No hashtag spam (3 max, only if relevant)

Video Script Outline

Create a video script outline for a [length]-minute video about [topic]. Structure: - Hook (first 5 seconds) - Problem statement - Key points (3 max) - CTA Target platform: [YouTube / LinkedIn / Instagram Reels] Speaker: [who's on camera]

Client Work

Start a Proposal

Follow the CDE Proposal Template Protocol. Claude knows the structure if your CLAUDE.md is set up.

I need to build a proposal for [client name]. Here's what I know: - Client: [name and what they do] - Services we're proposing: [list] - Budget range: [if known] - Timeline: [if known] Check the Proposal Template Protocol before starting. Build this as a GitHub Pages site following CDE standards.

Meeting Prep

I have a meeting with [client name] in [timeframe]. Help me prep: - What we've been working on with them: [brief context] - What I want to discuss: [topics] - What I need from them: [decisions, approvals, assets] Give me a one-page agenda and a list of questions to ask.

Campaign Performance Summary

If you have Google Ads MCP connected.

Pull performance data for [client name]'s Google Ads account for the last 30 days. I need: - Total spend, conversions, CPA - Top 5 campaigns by conversions - Any campaigns with CPA above [target] - Week-over-week trend Format as a summary I can paste into a client email.

Meeting Notes Cleanup

Tips for Better Results

Be Specific
"Write a blog post" → generic result
"Write an 800-word post about why hotels should track Scope 3 emissions, targeting sustainability directors" → useful result
Give Context
Claude doesn't know your client. Spend 2 sentences explaining who they are, what they do, and who their audience is. It pays off in every output.
Iterate, Don't Restart
If the first output is 70% right, say what's wrong and ask Claude to fix it. Don't start over with a new prompt — Claude remembers the full conversation.
Check the Protocols
Before proposals, shopping feeds, or content: tell Claude to check the relevant protocol page. It'll follow the CDE standard automatically.
