What Is This?

A plain-English explanation of what you're plugging into.

Claude Code in 30 Seconds

Claude Code is an AI assistant that lives in your terminal (or VS Code). It can read files, write code, run commands, search the web, talk to APIs, and automate workflows. Think of it as a very capable junior teammate that never sleeps.

You don't need to be a developer to use it. If you can describe what you want in plain English, Claude Code can usually do it.

What CDE Built on Top

Out of the box, Claude Code is a blank slate. CDE has layered a full operational system on top of it:

The Stack
CLAUDE.mdInstructions file that tells Claude who it is, what rules to follow, and how to behave. Every agent has one.
MCP ServersPlugins that give Claude new abilities — reading Google Drive, generating images, controlling a browser, pulling YouTube transcripts.
AgentsSpecialized Claude instances with specific jobs — blog briefings, transcript processing, content writing, image generation.
HooksSecurity guardrails that run before every command. They block dangerous operations and protect credentials.
Webhook ServerA deployed Vercel service that receives triggers from Zapier, WhatsApp, and other external tools to kick off agents automatically.

Three Layers

Understanding these layers helps you know what's shared vs. what's yours:

LayerWhat It IsShared?
InfrastructureAgents, MCP servers, hooks, CLAUDE.md files, webhook serverYes — via this repo
ConfigurationAPI keys, OAuth tokens, environment variablesNo — per person
StateConversation history, memory files, scratchpadNo — local only
You clone the Infrastructure layer from this repo. You set up your own Configuration. State is generated as you work.
